# laravel-messenger ‚Äî Routes & Controllers

## API Routes (prefix from config: default `/email`, middleware: `api`)

| Method | URI | Controller | Route Name |
|--------|-----|-----------|------------|
| GET | `/email/t/{hash}` | MailTrackingController@open | messenger.tracking.open |
| GET | `/email/n` | MailTrackingController@click | messenger.tracking.click |
| POST | `/email/sns` | MailTrackingSnsController@callback | messenger.tracking.sns |
| GET/POST | `/email/vonage-dlr` | VonageDlrController@callback | messenger.tracking.vonage-dlr |

### Tracking Routes

- **Open pixel** (`/email/t/{hash}`) ‚Äî returns 1x1 transparent GIF, increments `tracking_opens`, sets `tracking_opened_at` on first open, dispatches `MessageOpenedEvent`
- **Link click** (`/email/n?l=...&h=...`) ‚Äî validates signed URL, increments `tracking_clicks`, sets `tracking_clicked_at` on first click, dispatches `MessageLinkClickedEvent`, redirects to original URL
- **SNS callback** (`/email/sns`) ‚Äî receives SNS notifications (SubscriptionConfirmation + Notification), dispatches Record*Jobs based on event type (Delivery, Bounce, Complaint, Reject, Open, Click), fires `SesSnsWebhookReceivedEvent`
- **Vonage DLR** (`/email/vonage-dlr`) ‚Äî receives Vonage SMS delivery receipts, dispatches `RecordVonageDlrJob`. Gated by `config('messenger.tracking.vonage_dlr.enabled')` (default: `false`)

## Web Routes (prefix: `/emessenger/nova`, middleware: `web` + `signed`)

All web routes require a valid signature (generated by Nova actions with time-limited expiry).

| Method | URI | Purpose | Expiry |
|--------|-----|---------|--------|
| GET | `/nova/preview/{id}` | Preview message type template (renders a selected message through the type's mail class) | 10 min |
| GET | `/nova/preview-message/{message}` | View actual rendered HTML of a sent message (stored content) | 15 min |
| GET | `/nova/ses-sns-status` | Legacy SES/SNS status page | 30 min |
| GET | `/nova/ses-sns-dashboard` | SES/SNS dashboard (full health checks, DNS records, actions, AWS console links) | 30 min |
| POST | `/nova/ses-sns-dashboard/commands/{command}` | Execute setup/teardown/check commands from dashboard UI | ‚Äî |
| GET/POST | `/nova/ses-sns-dashboard/custom-mail-action` | Compose and send ad-hoc custom emails from dashboard | ‚Äî |

### Dashboard Commands (via POST `/commands/{command}`)

| Command | Maps to |
|---|---|
| `setup-tracking` | `SesSnsSetupService::setup()` |
| `check-tracking` | `SesSnsSetupService::check()` |
| `setup-sending` | `SesSendingSetupService::setup()` |
| `check-sending` | `SesSendingSetupService::check()` |
| `test-events` | `SesEventSimulatorService::simulate()` |
| `teardown` | `SesSnsSetupService::teardown()` |

## Controllers

- **MailTrackingController** (`src/Tracking/MailTrackingController.php`) ‚Äî handles open pixel & link click tracking with signature validation
- **MailTrackingSnsController** (`src/Tracking/MailTrackingSnsController.php`) ‚Äî receives SNS webhooks, handles SubscriptionConfirmation, dispatches Record*Jobs per event type
- **VonageDlrController** (`src/Http/Controllers/VonageDlrController.php`) ‚Äî receives Vonage SMS delivery receipts, dispatches `RecordVonageDlrJob` (gated by `tracking.vonage_dlr.enabled`)
- **SesSnsDashboardController** (`src/Nova/Controllers/SesSnsDashboardController.php`) ‚Äî renders SES/SNS dashboard with health checks, DNS records, identity details, AWS console links
- **SesSnsDashboardCommandController** (`src/Nova/Controllers/SesSnsDashboardCommandController.php`) ‚Äî executes setup/teardown/check/test commands from dashboard UI
